# Detecting Authorization Vulnerabilities via LLM-Assisted Semantic Access Control Modeling

We propose LLM4AVD, an LLM-driven approach for detecting authorization vulnerabilities. LLM4AVD introduces a unified access control model that captures high-level business semantics and diverse authorization mechanisms across source-code-level, framework-level, and database-level authorizations. Leveraging LLMs, LLM4AVD automatically extracts comprehensive access control models of the target software and identifies potential vulnerabilities by detecting semantic inconsistencies in access control behaviors.

## Benchmark

We construct a new dataset, AuthVul, which contains 230 historical authorization vulnerabilities extracted from representative open-source Java Web applications and 19 vulnerabilities extracted from industrial applications. Due to confidentiality constraints, the detailed vulnerability information of these industrial applications cannot be publicly released. The open-source data is available [here](https://https://github.com/LLM4AVD/LLM4AVD/blob/main/benchmark.json).

## Usage
- Download joern (based on [joern4.0.189](https://github.com/joernio/joern/releases/download/v4.0.189/joern-cli.zip))
- Create the Python environment
```bash
conda env create -f environment.yml
```
- Set API keys by creating a `.env` file in the root directory
```bash
# qwen api key
GPT_4o_mini_KEY=sk-xx
BASE_URL=xx
Qwen3_32B_KEY=""
Qwen3_32B_BASE_URL=""
```

### Step 1: Extract Sensitive Resources
```bash
export tool_path=""
python $tool_path/callchainTools/extract_resource.py \
    --project-name "project_name" \
    --root-dir /path/to/project \
    --output-dir /path/to/output
```
Parameter description:
- --project-name: Can be any name
- --root-dir: Path to the target code repository
- --output-dir: Output folder

### Step 2: Extract Call Chains
```bash
javasrc2cpg -J-Xmx10240m /path/to/project --output /path/to/output
python /callchainTools/extract_callchain.py \
    --project-name JeecgBoot-ff08336 \
    --resources-path /path/to/output/final_resources.json \
    --project-path /path/to/project \
    --cpg-path /path/to/output/cpg.bin \
    --out-dir /path/to/output
```
Parameter description:
- --project-name: Can be any name
- --resources-path: Path to the sensitive resource extraction result from Step 1
- --project-path: Path to the target code repository
- --cpg-path: Output generated by the `javasrc2cpg` command
- --output-dir: Output folder

### Step 3: Generate the Access Control Model
```bash
python /AccessControl/extract_access_control_model.py \
    --model_name gpt-4o-mini \
    --resources_path /path/to/project/final_resources.json \
    --call_chains_path /path/to/output/{project_name}_call_chains_up_down.json \
    --output_dir /path/to/output 
```
Parameter description:
- --model_name: Name of the model to use
- --resources_path: Path to the sensitive resource extraction result from Step 1
- --call_chains_path: Call chain output from Step 2
- --output-dir: Output folder

### Step 4: Consistency Analysis for Vulnerability Detection
```bash
python /AccessControl/vul_detection.py \
    --model_name gpt-4o-mini \
    --access_control_model_path /path/to/output/{model_name}.json \
    --processed_access_control_model_path /path/to/output/processed_access_control_model.json\
    --complete_access_control_model_path /path/to/output/complete_access_control_model.json\
    --diff_access_control_model_path /path/to/output/diff_access_control_model.json\
    --call_chains_path /path/to/{project_name}_call_chains_up_down.json\
    --final_result_path /path/to/output/final_result.json\
    --false_positives_path /path/to/output/false_positives.json \
    --resources_path /path/to/output/final_resources.json
```
Parameter description:
- --model_name: Model name
- --access_control_model_path: Output result from Step 3
- --processed_access_control_model_path: Path for intermediate output
- --complete_access_control_model_path: Path for intermediate output
- --diff_access_control_model_path: Path for intermediate output
- --call_chains_path: Output result from Step 2
- --final_result_path: Final output path
- --false_positives_path: False-positive mitigation result path
- --resources_path: Output result path from Step 1
